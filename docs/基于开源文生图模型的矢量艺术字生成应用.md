# 基于开源文生图模型的矢量艺术字生成应用（赛题技术需求文档）

## 1. 项目任务概述
本项目旨在基于开源文生图模型制作一个可本地部署的艺术字生成应用。系统需构建完整的 ComfyUI 工作流，根据用户输入的文字内容与风格提示词生成对应的艺术字图像；同时，结合 Python 程序对生成的图像进行预处理、轮廓提取、颜色分层与路径拟合，将位图艺术字转换为可编辑、可缩放的矢量图像，并输出符合标准规范的 SVG 矢量文件。

## 2. 项目目标
项目内容和实施核心包含以下五点：
1. **跨文本生成支撑**：基于开源文生图模型搭建 ComfyUI 工作流，支持中文、英文、数字及符号等艺术字图像的高质量生成。
2. **多维属性可控性**：支持通过提示词精准控制文字内容、字体风格、配色、装饰元素、版式结构与背景透明等效果。
3. **高可读性与结构保真**：生成图像应保证文字主体清晰可辨、结构完整、艺术风格统一，严格避免缺字、错字、笔画断裂和主体遮挡等常见缺陷。
4. **多层级矢量化转换**：基于 Python 程序完成图像矢量化，精准保留主文字轮廓、装饰图形、描边、阴影或渐变等主要视觉特征。
5. **标准规范化输出**：输出标准 SVG 矢量图及配套的 PNG 预览图、参数配置文件和运行日志，确保所有生成结果可编辑、可复现、可批量生成。

## 3. 艺术字生成与矢量化对象定义标准

| 对象类别 | 定义说明 | 包含内容 | 输出要求 |
| :--- | :--- | :--- | :--- |
| **提示词输入** | 用户输入的文字内容与艺术风格描述 | 中文/英文/数字/符号、色彩、材质、装饰元素、版式、用途场景、负面提示词等 | 需结构化记录，支持完全复现生成过程 |
| **艺术字图像** | 由 ComfyUI 工作流根据提示词生成的位图结果 | 主文字、描边、填充、阴影、装饰图案、留白或透明背景 | 输出 PNG/JPG（推荐透明 PNG），同时保存内嵌生成参数 |
| **矢量艺术字** | 通过 Python 程序将位图轮廓转换得到的矢量结果 | 主文字路径、装饰路径、颜色块、分组信息、画布尺寸与视口信息 | 输出标准 SVG 格式，路径必须闭合、可自由缩放且可在专业矢量软件中编辑 |
| **预览/元数据** | 用于自动化验收和追溯的可视化与结构化文件 | PNG 预览图、JSON 元数据、系统运行日志、批量任务结果清单 | 保持严格的文件命名一致性，便于自动化脚本核验 |

## 4. 项目需求

### 4.1 核心功能要求

#### （1）开源文生图模型与 ComfyUI 工作流搭建（前置要求）
* **本地化与合规性**：系统应使用可本地部署、许可合规的开源文生图模型作为基座，不得依赖任何闭源商业 API 完成核心生成任务。
* **工作流节点覆盖**：需在 ComfyUI 中构建完整且结构清晰的工作流，完整覆盖模型加载、提示词解析、采样生成、高清修复、透明背景处理、结果保存等核心节点。
* **参数化可配置**：工作流参数应支持动态配置，包括随机种子、分辨率、采样步数、CFG/引导强度，并支持可选挂载风格 LoRA、ControlNet、字体参考图等扩展模块。
* **可复现交付**：工作流应支持导出为 ComfyUI 可直接导入运行的 API/Dev 格式 JSON 文件，并附带节点说明和详细的模型依赖清单。

#### （2）艺术字内容可控生成（核心任务一）
* **文本一致性约束**：系统应根据用户输入的文字内容生成对应艺术字图像，文字内容必须与提示词及输入参数保持完全一致。
* **应用场景覆盖**：支持中文不少于 2—8 个汉字、英文单词/短语、数字比例或促销信息等常见艺术字应用场景。
* **容错与修正机制**：需大幅降低缺字、错字、乱码、笔画粘连、文字变形过度等模型原生问题；对失败样例应给出可重复生成或参数修正的有效机制。
* **负面提示词排除**：支持负面提示词（Negative Prompt）约束，有效避免无关人物、杂乱背景、错误干扰文字、低清晰度、严重畸变等视觉干扰。

#### （3）风格与版式控制（核心任务二）
* **丰富风格矩阵**：支持多种主流艺术字风格，包括但不限于国风书法、清新植物、海洋浪漫、促销海报、卡通描边、金属质感、霓虹灯、节日主题等。
* **视觉特征解耦**：支持精确控制主色、辅色、描边、阴影、渐变、材质纹理、装饰元素和整体视觉构图。
* **多样化版式**：可按照横排、竖排、弧形、上下组合、主副标题组合等版式进行布局生成，保证主体居中、比例协调、留白合理。
* **图文层次分明**：对于包含数字或英文副标题的混合艺术字，应支持主文字与副文字层次清晰、空间排布合理。

#### （4）图像预处理与透明背景输出（核心任务三）
* **统一规范输出**：生成结果应支持统一分辨率输出，默认不低于 1024×1024 像素，或按需输出 16:9、1:1、3:2 等多标准比例。
* **图像预处理流水线**：应对生成的位图进行背景分离、边缘去噪、主体裁剪、颜色量化、抗锯齿保留等系列预处理。
* **高质量透明通道**：需输出带 Alpha 通道的背景透明 PNG，并保证主体边缘自然过渡，无明显白边、黑边或残留噪点。
* **自动化目录结构**：批量任务中应自动建立标准规范的输出目录，按照任务编号、提示词摘要、随机种子等结构化信息命名结果文件。

#### （5）Python 艺术字矢量化转换（核心任务四）
* **自动化管道处理**：Python 程序应能独立读取艺术字 PNG/JPG 图像，自动化完成主体分割、轮廓检测、连通域分析、颜色分层与路径拟合。
* **视觉层级重构**：需将主文字、装饰图形、描边、阴影等主要视觉区域转换为对应的 SVG Path 或 Shape 元素，并尽可能保留原始的视觉叠加层级和颜色信息。
* **路径几何质量**：生成的矢量路径应保证闭合、平滑、可无限缩放，避免路径过度碎片化、自交、笔画断裂、局部空洞丢失或过度简化。
* **开放超参数调节**：应提供灵活的矢量化参数配置接口，包括颜色聚类数量（K-Means）、平滑阈值、最小连通区域过滤面积、路径拟合精度、是否保留渐变/阴影等。
* **闭环核验**：系统需自动生成 SVG 预览图或回渲染 PNG，用于对比原始艺术字图像，以便自动化验收核验。

#### （6）应用交互与结果管理（核心任务五）
* **简洁人机交互**：应用应提供简洁直观的本地操作界面（GUI）或命令行（CLI）入口，并无缝支持单条提示词生成、批量提示词生成以及已有外部图片矢量化三种独立模式。
* **参数全曝光**：用户可在界面或命令中自由调配文字内容、风格提示词、负面提示词、分辨率、输出格式、随机种子及矢量化转换核心参数。
* **全生命周期日志**：系统应结构化记录生成全过程，包含模型版本、工作流版本、关键超参数、各模块处理耗时、异常/错误信息以及最终输出路径。
* **完整交付产物包**：每个任务的结果文件组合应至少包含：原始生成图、透明背景 PNG、SVG 矢量图、SVG 回渲染预览图、JSON 元数据配置文件和运行日志文件。

### 4.2 性能与可靠性要求
1. **文本一致性指标**：在标准测试集（≥100 条艺术字提示词，覆盖中、英、数及混合文本）上，文字内容生成正确率 $\ge 90\%$，严重错字/漏字率 $\le 10\%$。
2. **视觉质量指标**：艺术字主体完整率 $\ge 95\%$，主体清晰可辨，整体视觉风格与提示词匹配率 $\ge 85\%$，不得出现破坏字形的严重噪声、主体非正常裁切或明显无关干扰元素。
3. **矢量还原度指标**：SVG 回渲染预览图与透明 PNG 在主体区域的轮廓覆盖率 $\ge 95\%$，主要轮廓平均偏差 $\le 2$ 像素（按 1024px 长边归一化测试）。
4. **路径可编辑性**：主文字与主要装饰元素应按颜色块或连通域进行科学分组（`<g>` 标签），输出的 SVG 文件必须能被 Inkscape、Adobe Illustrator 等主流专业矢量软件正常打开、拆分、改色和编辑。
5. **处理效率指标**：单张 1024×1024 像素位图图片的矢量化处理耗时 $\le 10$ 秒；单条提示词从发起生成到最终矢量化文件输出的端到端总耗时 $\le 90$ 秒（测试标准环境：具备 12GB 显存的主流 GPU / 32GB RAM / SSD）。
6. **批量稳定性**：系统应支持连续稳定处理 100 条以上提示词而无程序崩溃；在遇到异常提示词、模型缺失、显存不足（OOM）或输入图片损坏时，系统应能给出友好的用户提示并准确写入错误日志。
7. **软硬件兼容性**：支持 Windows 10/11 操作系统，兼容常见 NVIDIA GPU 驱动及 CUDA 环境；在无 GPU 环境下，系统可自动降级运行，支持独立运行矢量化模块，并给出生感模块不可用的友好提示。
8. **安全性与隐私**：所有提示词、临时图片和最终矢量结果默认在本地完成处理，严禁未经授权静默上传至第三方服务器；操作日志本地保留应不少于 180 天。

## 5. 技术要求

### 5.1 开发语言与框架
* 使用 **Python 3.8** 或更高版本开发应用主体工程、图像预处理流水线、矢量化转换核心算法、文件管理模块和自动化测试脚本。
* 使用 **ComfyUI** 搭建核心文生图工作流，其模型推理、节点配置、参数保存和批量远程调用（通过 API 形式）应具备严格的可复现性。
* 允许且推荐使用 `PyTorch`、`OpenCV (cv2)`、`Pillow (PIL)`、`NumPy`、`scikit-image`、`svgwrite`、`potrace` / `vtracer` 类开源图像处理与矢量化工具或同等能力组件，但必须确保开源许可合规（如 MIT、Apache 2.0 或 BSD）。

### 5.2 模型与工作流要求
* 系统的基座模型（如 SDXL、Flux 等）、LoRA 权重、ControlNet 结构控制模型、字体参考模型或其他自定义插件节点均需在交付文档中显式指明来源、精确版本、开源许可说明和详细的本地部署激活指引。
* ComfyUI 工作流内部必须包含清晰的节点分组、命名、中文/英文备注与色彩标示，便于评审人员导入后快速理清逻辑并复现结果。
* 严禁将核心生成能力外包给联机闭源第三方接口。若使用了第三方开源自定义节点，需在交付清单和依赖中明确标明安装方式（如通过 ComfyUI-Manager 或 Git 仓库地址）和依赖版本。
* 系统应通过代码级控制，支持固定随机种子（Seed）以完全复现实验级图像结果，并支持批量自动化读取 CSV/JSON/TXT 格式的提示词列表文件。

### 5.3 矢量化处理要求
* 矢量化程序应优先保留艺术字的主体几何结构和视觉可编辑性，**绝对禁止**仅输出一层包裹了 Base64 位图编码的虚假 SVG 文件（即禁止直接内嵌 `<image>` 标签作为路径结果）。
* 导出的标准 SVG 代码中应严格包含 `viewBox` 属性、确切的画布尺寸、逻辑清晰的路径分组（`<g>`）、精准的颜色填充（`fill`）、透明背景设置和必要的生成元数据（Metadata）。
* 对于多色复合艺术字，程序应具备出色的颜色分层或聚类能力；对于文字本身的描边（Stroke）、阴影（Shadow）、特定外挂装饰元素，应尽可能作为独立的路径对象或分组进行解耦输出。
* 输出的 SVG 源码应百分之百通过标准的 XML 基础语法校验，能够被主流现代浏览器（Chrome, Edge, Safari）渲染，并能完美兼容商业矢量设计软件。

## 7. 代码质量要求
* **高注释率约束**：代码工程的整体注释率必须 $\ge 25\%$。对于图像预处理（如自适应二值化、形态学滤波）、轮廓提取、路径拟合（Bezier 曲线控制点计算）、颜色分层（如 K-Means 聚类）与回渲染对比等核心算法模块，必须在源码头部附加原理和数学实现说明。
* **模块化高内聚**：代码工程结构设计清晰合理，在物理目录和逻辑架构上，必须至少明确区分以下模块：
  * 配置模块（Config Management）
  * ComfyUI 驱动调用模块（ComfyUI API Client）
  * 图像预处理模块（Image Preprocessing）
  * 核心矢量化转换模块（Vectorization Engine）
  * 结果管理与 IO 模块（Result Management）
  * 自动化功能测试模块（Testing Framework）
* **依赖确定性**：提供完备、无版本冲突的 `requirements.txt` 或 Anaconda 的 `environment.yml` 配置文件，确保评审人员在干净隔离的环境中可一键复现式安装。
* **自动化测试配套**：随工程配套提供高覆盖率的自动化测试脚本，全面覆盖单图矢量化基准测试、百条提示词批量生成并发测试、损坏或越界等异常输入测试以及生成的 SVG XML 语法合法性校验等丰富场景。
